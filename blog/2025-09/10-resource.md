# 性能优化

## 资源加载优化

关键资源（如CSS、JS、图片）过大或加载顺序不合理，导致首屏内容渲染延迟。
使用rel="preload", async, defer 属性来调整资源加载策略
资源合并与压缩, 使用webp, 图片懒加载

### 对比preload, async, defer 

特性	rel="preload"	async	defer
是否下载时阻塞解析	❌ 否	❌ 否	❌ 否
下载完成后是否立即执行	❌ 否（需手动引入）	✅ 是	❌ 否（等解析完成）
执行是否阻塞解析	N/A	✅ 是	❌ 否
执行顺序	手动控制	不保证（谁快谁先）	保证（按书写顺序）
适用资源类型	所有（JS/CSS/Font等）	仅 JS	仅 JS
是否执行脚本	❌ 仅下载	✅ 是	✅ 是
五、最佳实践建议
关键 JS（首屏逻辑）：
使用 rel="preload" + <script> 引入，确保尽早下载。
或使用 defer，保证在 DOM 就绪后执行。
非关键 JS（统计、广告）：
使用 async，避免阻塞且无需关心执行顺序。
有依赖关系的 JS 模块：
使用 defer，保证执行顺序。
关键字体/图片：
使用 rel="preload" 提升加载优先级。
